# 사전 훈련된 모델 공유하기 (Sharing pretrained models)

<Youtube id="9yY3RB_GSPM"/>

아래 단계에서는 사전 훈련된 모델을 🤗 Hub에 공유하는 가장 쉬운 방법을 살펴보겠습니다. Hub에 직접 모델을 공유하고 업데이트하는 것을 간단하게 만드는 도구와 유틸리티가 있으며, 이를 아래에서 탐구할 것입니다.

모델을 훈련하는 모든 사용자가 커뮤니티와 공유하여 기여하도록 권장합니다. 모델을 공유하는 것은 매우 구체적인 데이터셋으로 훈련된 경우에도 다른 사람들에게 도움이 되어 시간과 컴퓨팅 리소스를 절약하고 유용한 훈련된 아티팩트에 접근할 수 있도록 제공합니다. 결국, 당신은 다른 사람들이 수행한 작업으로부터 이익을 얻을 수 있습니다!

새 모델 저장소를 생성하는 방법은 세 가지가 있습니다:

- **`push_to_hub`** API 사용
- **`huggingface_hub`** Python 라이브러리 사용
- 웹 인터페이스 사용

저장소를 생성한 후에는 git 및 git-lfs를 통해 파일을 업로드할 수 있습니다. 다음 섹션에서는 모델 저장소를 생성하고 파일을 업로드하는 과정을 안내할 것입니다.


## `push_to_hub` API 사용하기 (Using the `push_to_hub` API)

<Youtube id="Zh0FfmVrKX0"/>

Hub에 파일을 업로드하는 가장 간단한 방법은 **`push_to_hub`** API를 활용하는 것입니다.

더 나아가기 전에, **`huggingface_hub`** API가 당신이 누구인지, 그리고 어떤 네임스페이스에 쓰기 접근 권한이 있는지 알 수 있도록 인증 토큰을 생성해야 합니다. **`transformers`**가 설치된 환경에 있는지 확인하십시오([Setup](/course/chapter0) 참조). 노트북 환경에 있다면 다음 함수를 사용하여 로그인할 수 있습니다:

```python
from huggingface_hub import notebook_login

notebook_login()
```

터미널에서는 다음을 실행할 수 있습니다:

```bash
huggingface-cli login
```

두 경우 모두 Hub에 로그인하는 데 사용하는 것과 동일한 사용자 이름과 암호를 입력하라는 메시지가 표시됩니다. 아직 Hub 프로필이 없으면 [여기](https://huggingface.co/join)에서 생성해야 합니다.

좋습니다! 이제 캐시 폴더에 인증 토큰이 저장되었습니다. 이제 저장소를 만들어 봅시다!

`Trainer` API를 사용하여 모델을 훈련했다면, Hub에 모델을 업로드하는 가장 쉬운 방법은 **`TrainingArguments`**를 정의할 때 **`push_to_hub=True`**를 설정하는 것입니다:

```py
from transformers import TrainingArguments

training_args = TrainingArguments(
    "bert-finetuned-mrpc", save_strategy="epoch", push_to_hub=True
)
```

**`trainer.train()`**을 호출하면, `Trainer`는 모델이 저장될 때마다(여기서는 에폭마다) 당신의 네임스페이스에 있는 저장소에 모델을 Hub에 업로드할 것입니다. 이 저장소는 당신이 선택한 출력 디렉토리(여기서는 **`bert-finetuned-mrpc`**)와 이름이 같지만, **`hub_model_id = "a_different_name"`**을 사용하여 다른 이름을 선택할 수 있습니다.

당신이 회원인 조직에 모델을 업로드하려면, **`hub_model_id = "my_organization/my_repo_name"`**으로 전달하기만 하면 됩니다.

훈련이 완료되면, 모델의 최종 버전을 업로드하기 위해 최종 **`trainer.push_to_hub()`**를 실행해야 합니다. 또한 사용된 하이퍼파라미터 및 평가 결과를 보고하는 모든 관련 메타데이터가 포함된 모델 카드를 생성할 것입니다! 다음은 그러한 모델 카드에서 찾을 수 있는 콘텐츠의 예시입니다:

<div class="flex justify-center">
  <img src="https://huggingface.co/datasets/huggingface-course/documentation-images/resolve/main/en/chapter4/model_card.png" alt="An example of an auto-generated model card." width="100%"/>
</div>

더 낮은 수준에서, Model Hub에 접근하는 것은 모델, 토크나이저 및 구성 객체에서 **`push_to_hub()`** 메서드를 통해 직접 수행할 수 있습니다. 이 메서드는 저장소 생성과 모델 및 토크나이저 파일을 저장소에 직접 푸시하는 것을 모두 처리합니다. 아래에서 보게 될 API와 달리 수동 처리가 필요하지 않습니다.

작동 방식을 파악하기 위해 먼저 모델과 토크나이저를 초기화해 봅시다:

```py
from transformers import AutoModelForMaskedLM, AutoTokenizer

checkpoint = "camembert-base"

model = AutoModelForMaskedLM.from_pretrained(checkpoint)
tokenizer = AutoTokenizer.from_pretrained(checkpoint)
```

당신은 이것들로 원하는 모든 것을 자유롭게 할 수 있습니다. 토크나이저에 토큰을 추가하거나, 모델을 훈련하거나, 미세 조정할 수 있습니다. 결과 모델, 가중치 및 토크나이저에 만족하면 **`model`** 객체에서 직접 사용할 수 있는 **`push_to_hub()`** 메서드를 활용할 수 있습니다:

```py
model.push_to_hub("dummy-model")
```

이렇게 하면 당신의 프로필에 새 저장소 **`dummy-model`**이 생성되고 모델 파일로 채워집니다.
토크나이저에도 동일하게 적용하여 모든 파일이 이제 이 저장소에서 사용할 수 있도록 합니다:

```py
tokenizer.push_to_hub("dummy-model")
```

조직에 속해 있다면, **`organization`** 인수를 지정하여 해당 조직의 네임스페이스에 업로드하면 됩니다:

```py
tokenizer.push_to_hub("dummy-model", organization="huggingface")
```

특정 Hugging Face 토큰을 사용하려면, **`push_to_hub()`** 메서드에 이를 지정할 수 있습니다:

```py
tokenizer.push_to_hub("dummy-model", organization="huggingface", use_auth_token="<TOKEN>")
```

이제 Model Hub로 이동하여 새로 업로드된 모델을 찾으십시오: *https://huggingface.co/user-or-organization/dummy-model*.

"Files and versions" 탭을 클릭하면 다음 스크린샷에 보이는 파일을 볼 수 있습니다:

<div class="flex justify-center">
<img src="https://huggingface.co/datasets/huggingface-course/documentation-images/resolve/main/en/chapter4/push_to_hub_dummy_model.png" alt="Dummy model containing both the tokenizer and model files." width="80%"/>
</div>

<Tip>

✏️ **직접 해보세요!** **`bert-base-cased`** 체크포인트와 연결된 모델 및 토크나이저를 가져와 **`push_to_hub()`** 메서드를 사용하여 당신의 네임스페이스에 있는 저장소에 업로드하십시오. 삭제하기 전에 저장소가 당신의 페이지에 올바르게 나타나는지 다시 한 번 확인하십시오.

</Tip>

보셨듯이, **`push_to_hub()`** 메서드는 여러 인수를 허용하므로 특정 저장소 또는 조직 네임스페이스에 업로드하거나 다른 API 토큰을 사용하는 것이 가능합니다. 가능한 것이 무엇인지 파악하기 위해 [🤗 Transformers 문서](https://huggingface.co/transformers/model_sharing)에서 직접 사용 가능한 메서드 사양을 살펴보는 것을 권장합니다.

**`push_to_hub()`** 메서드는 Hugging Face Hub에 대한 직접적인 API를 제공하는 **`huggingface_hub`** Python 패키지를 기반으로 합니다. 이 패키지는 🤗 Transformers 및 [`allenlp`](https://github.com/allenai/allennlp)와 같은 기타 여러 기계 학습 라이브러리에 통합되어 있습니다. 이 장에서는 🤗 Transformers 통합에 중점을 두지만, 자신의 코드 또는 라이브러리에 통합하는 것은 간단합니다.

새로 생성된 저장소에 파일을 업로드하는 방법을 보려면 마지막 섹션으로 이동하십시오!

## `huggingface_hub` Python 라이브러리 사용하기 (Using the `huggingface_hub` Python library)

**`huggingface_hub`** Python 라이브러리는 모델 및 데이터셋 허브를 위한 일련의 도구를 제공하는 패키지입니다. 이는 허브의 저장소에 대한 정보를 얻고 관리하는 것과 같은 일반적인 작업을 위한 간단한 메서드와 클래스를 제공합니다. 이는 해당 저장소의 콘텐츠를 관리하고 Hub를 프로젝트 및 라이브러리에 통합하기 위해 git 위에 작동하는 간단한 API를 제공합니다.

**`push_to_hub`** API를 사용하는 것과 마찬가지로, 캐시에 API 토큰을 저장해야 합니다. 이를 위해서는 이전 섹션에서 언급한 CLI의 **`login`** 명령을 사용해야 합니다(다시 말하지만, Google Colab에서 실행하는 경우 이 명령 앞에 **`!`** 문자를 붙여야 합니다):

```bash
huggingface-cli login
```

**`huggingface_hub`** 패키지는 우리의 목적에 유용한 여러 메서드와 클래스를 제공합니다. 첫째, 저장소 생성, 삭제 및 기타 관리를 위한 몇 가지 메서드가 있습니다:

```python no-format
from huggingface_hub import (
    # 사용자 관리
    login,
    logout,
    whoami,

    # 저장소 생성 및 관리
    create_repo,
    delete_repo,
    update_repo_visibility,

    # 그리고 콘텐츠에 대한 정보 검색/변경을 위한 일부 메서드
    list_models,
    list_datasets,
    list_metrics,
    list_repo_files,
    upload_file,
    delete_file,
)
```


또한, 로컬 저장소를 관리하는 매우 강력한 **`Repository`** 클래스를 제공합니다. 다음 몇 섹션에서 이를 활용하는 방법을 이해하기 위해 이러한 메서드와 클래스를 탐색할 것입니다.

**`create_repo`** 메서드를 사용하여 허브에 새 저장소를 만들 수 있습니다:

```py
from huggingface_hub import create_repo

create_repo("dummy-model")
```

이렇게 하면 당신의 네임스페이스에 **`dummy-model`** 저장소가 생성됩니다. 원한다면 **`organization`** 인수를 사용하여 저장소가 속할 조직을 지정할 수 있습니다:

```py
from huggingface_hub import create_repo

create_repo("dummy-model", organization="huggingface")
```

이는 당신이 해당 조직에 속해 있다고 가정하고 **`huggingface`** 네임스페이스에 **`dummy-model`** 저장소를 생성할 것입니다.
유용할 수 있는 다른 인수는 다음과 같습니다:

- **`private`**, 저장소가 다른 사람에게 표시되어야 하는지 여부를 지정합니다.
- **`token`**, 캐시에 저장된 토큰을 주어진 토큰으로 재정의하려는 경우.
- **`repo_type`**, 모델 대신 **`dataset`** 또는 **`space`**를 생성하려는 경우. 허용되는 값은 **`"dataset"`** 및 **`"space"`**입니다.

저장소가 생성되면 파일을 추가해야 합니다! 이를 처리할 수 있는 세 가지 방법을 보려면 다음 섹션으로 이동하십시오.

## 모델 파일 업로드하기 (Uploading the model files)

Hugging Face Hub에서 파일을 관리하는 시스템은 일반 파일의 경우 **git**을 기반으로 하고, 더 큰 파일의 경우 **git-lfs**([Git Large File Storage](https://git-lfs.github.com/)의 약자)를 기반으로 합니다.

다음 섹션에서는 Hub에 파일을 업로드하는 세 가지 다른 방법, 즉 **`huggingface_hub`**를 통한 방법과 git 명령을 통한 방법을 다룰 것입니다.

### `upload_file` 접근 방식 (The `upload_file` approach)

**`upload_file`**을 사용하려면 시스템에 git 및 git-lfs가 설치되어 있을 필요가 없습니다. HTTP POST 요청을 사용하여 파일을 🤗 Hub에 직접 푸시합니다. 이 접근 방식의 한계는 5GB보다 큰 파일은 처리할 수 없다는 것입니다.
파일이 5GB보다 큰 경우 아래에 자세히 설명된 다른 두 가지 방법을 따르십시오.

API는 다음과 같이 사용될 수 있습니다:

```py
from huggingface_hub import upload_file

upload_file(
    "<path_to_file>/config.json",
    path_in_repo="config.json",
    repo_id="<namespace>/dummy-model",
)
```

이렇게 하면 `<path_to_file>`에 있는 `config.json` 파일이 **`dummy-model`** 저장소의 루트에 `config.json`으로 업로드됩니다.
유용할 수 있는 다른 인수는 다음과 같습니다:

- **`token`**, 캐시에 저장된 토큰을 주어진 토큰으로 재정의하려는 경우.
- **`repo_type`**, 모델 대신 **`dataset`** 또는 **`space`**에 업로드하려는 경우. 허용되는 값은 **`"dataset"`** 및 **`"space"`**입니다.


### `Repository` 클래스 (The `Repository` class)

**`Repository`** 클래스는 git과 유사한 방식으로 로컬 저장소를 관리합니다. 이는 필요한 모든 기능을 제공하기 위해 git에서 발생할 수 있는 대부분의 문제점을 추상화합니다.

이 클래스를 사용하려면 git 및 git-lfs가 설치되어 있어야 하므로, 시작하기 전에 git-lfs가 설치되어 있는지 확인하고([여기](https://git-lfs.github.com/)에서 설치 지침 참조) 설정되었는지 확인하십시오.

방금 생성한 저장소를 가지고 놀기 시작하려면, 원격 저장소를 복제하여 로컬 폴더로 초기화하는 것으로 시작할 수 있습니다:

```py
from huggingface_hub import Repository

repo = Repository("<path_to_dummy_folder>", clone_from="<namespace>/dummy-model")
```

이렇게 하면 작업 디렉토리에 **`<path_to_dummy_folder>`** 폴더가 생성됩니다. 이 폴더에는 **`create_repo`**를 통해 저장소를 인스턴스화할 때 생성된 유일한 파일인 **`.gitattributes`** 파일만 포함되어 있습니다.

이 시점부터는 여러 전통적인 git 메서드를 활용할 수 있습니다:

```py
repo.git_pull()
repo.git_add()
repo.git_commit()
repo.git_push()
repo.git_tag()
```

그리고 다른 것들도 있습니다! 사용 가능한 모든 메서드의 개요를 보려면 [여기](https://github.com/huggingface/huggingface_hub/tree/main/src/huggingface_hub#advanced-programmatic-repository-management)에서 **`Repository`** 문서를 살펴보는 것을 권장합니다.

현재, 우리는 Hub에 푸시하려는 모델과 토크나이저를 가지고 있습니다. 우리는 저장소를 성공적으로 복제했으며, 따라서 해당 저장소 내에 파일을 저장할 수 있습니다.

먼저 최신 변경 사항을 풀(pull)하여 로컬 복제본이 최신 상태인지 확인합니다:

```py
repo.git_pull()
```

이 작업이 완료되면 모델 및 토크나이저 파일을 저장합니다:

```py
model.save_pretrained("<path_to_dummy_folder>")
tokenizer.save_pretrained("<path_to_dummy_folder>")
```

이제 **`<path_to_dummy_folder>`**에는 모든 모델 및 토크나이저 파일이 포함되어 있습니다. 파일을 스테이징 영역에 추가하고, 커밋하고, Hub에 푸시하는 일반적인 git 워크플로를 따릅니다:

```py
repo.git_add()
repo.git_commit("Add model and tokenizer files")
repo.git_push()
```

축하합니다! 방금 Hub에 첫 번째 파일을 푸시했습니다.

### git 기반 접근 방식 (The git-based approach)

이것은 파일을 업로드하는 가장 기본적인 접근 방식입니다. **git** 및 **git-lfs**를 통해 직접 수행할 것입니다. 대부분의 어려움은 이전 접근 방식으로 추상화되지만, 다음 메서드에는 몇 가지 주의 사항이 있으므로 더 복잡한 사용 사례를 따를 것입니다.

이 클래스를 사용하려면 git 및 git-lfs가 설치되어 있어야 하므로, 시작하기 전에 [git-lfs](https://git-lfs.github.com/)가 설치되어 있는지 확인하고(설치 지침은 여기 참조) 설정되었는지 확인하십시오.

먼저 git-lfs를 초기화하는 것으로 시작합니다:

```bash
git lfs install
```

```bash
Updated git hooks.
Git LFS initialized.
```

이 작업이 완료되면 첫 번째 단계는 모델 저장소를 복제하는 것입니다:

```bash
git clone [https://huggingface.co/](https://huggingface.co/)<namespace>/<your-model-id>
```

내 사용자 이름은 **`lysandre`**이고 모델 이름은 **`dummy`**를 사용했으므로, 나에게는 명령이 다음과 같이 보입니다:

```
git clone [https://huggingface.co/lysandre/dummy](https://huggingface.co/lysandre/dummy)
```

이제 작업 디렉토리에 *dummy*라는 폴더가 있습니다. 폴더로 **`cd`**하여 내용을 볼 수 있습니다:

```bash
cd dummy && ls
```

```bash
README.md
```

Hugging Face Hub의 **`create_repo`** 메서드를 사용하여 저장소를 방금 생성했다면, 이 폴더에는 숨겨진 **`.gitattributes`** 파일만 포함되어야 합니다. 웹 인터페이스를 사용하여 저장소를 생성하는 방법에 대한 이전 섹션의 지침을 따랐다면, 여기에 표시된 대로 폴더에는 숨겨진 **`.gitattributes`** 파일과 함께 단일 *README.md* 파일이 포함되어야 합니다.

구성 파일, 어휘 파일 또는 기본적으로 몇 메가바이트 미만의 모든 파일과 같은 일반 크기 파일을 추가하는 것은 모든 git 기반 시스템에서 하는 것과 똑같이 수행됩니다. 그러나 더 큰 파일은 **`huggingface.co`**로 푸시하기 위해 **git-lfs**를 통해 등록해야 합니다.

더미 저장소에 커밋하려는 모델과 토크나이저를 생성하기 위해 잠시 Python으로 돌아갑니다:

```py
from transformers import AutoModelForMaskedLM, AutoTokenizer

checkpoint = "camembert-base"

model = AutoModelForMaskedLM.from_pretrained(checkpoint)
tokenizer = AutoTokenizer.from_pretrained(checkpoint)

# 모델로 원하는 작업을 수행하고, 훈련하고, 미세 조정합니다...

model.save_pretrained("<path_to_dummy_folder>")
tokenizer.save_pretrained("<path_to_dummy_folder>")
```

이제 일부 모델 및 토크나이저 아티팩트를 저장했으므로, *dummy* 폴더를 다시 살펴보겠습니다:

```bash
ls
```

```bash
config.json  pytorch_model.bin  README.md  sentencepiece.bpe.model  special_tokens_map.json tokenizer_config.json  tokenizer.json
```

파일 크기를 확인하면(예: **`ls -lh`** 사용), 모델 상태 사전 파일(*pytorch_model.bin*)만 400MB가 넘는 유일한 예외임을 알 수 있습니다.

<Tip>
✏️ 웹 인터페이스에서 저장소를 생성할 때, *.bin* 및 *.h5*와 같은 특정 확장자를 가진 파일을 큰 파일로 간주하도록 *.gitattributes* 파일이 자동으로 설정되며, git-lfs는 당신 측의 설정 없이 이들을 추적합니다.
</Tip> 

이제 전통적인 Git 저장소에서 일반적으로 하는 것처럼 진행할 수 있습니다. **`git add`** 명령을 사용하여 모든 파일을 Git의 스테이징 환경에 추가할 수 있습니다:

```bash
git add .
```
