## 챕터 마무리 퀴즈

본 챕터에서 학습한 내용을 테스트해 봅시다!

1. **Hub의 모델은 무엇으로 제한됩니까?**
    1. 🤗 Transformers 라이브러리의 모델로 제한됩니다.
    2. 🤗 Transformers와 유사한 인터페이스를 가진 모든 모델로 제한됩니다.
    3. 제한이 없습니다.
    4. NLP와 어떤 식으로든 관련된 모델로 제한됩니다.

2. **Hub에서 모델을 어떻게 관리할 수 있습니까?**
    1. GCP 계정을 통해 관리할 수 있습니다.
    2. P2P(peer-to-peer) 배포를 통해 관리할 수 있습니다.
    3. Git 및 Git-LFS를 통해 관리할 수 있습니다.

3. **Hugging Face Hub 웹 인터페이스를 사용하여 무엇을 할 수 있습니까?**
    1. 기존 저장소(repository)를 포크(Fork)할 수 있습니다.
    2. 새 모델 저장소를 생성할 수 있습니다.
    3. 파일을 관리하고 편집할 수 있습니다.
    4. 파일을 업로드할 수 있습니다.
    5. 버전 간의 차이(diffs)를 볼 수 있습니다.

4. **모델 카드(model card)란 무엇입니까?**
    1. 모델에 대한 대략적인 설명이므로 모델 및 토크나이저 파일보다 덜 중요합니다.
    2. 재현성(reproducibility), 재사용성(reusability), 공정성(fairness)을 보장하는 방법입니다.
    3. 모델에 대한 정보를 검색하기 위해 실행할 수 있는 파이썬 파일입니다.

5. **🤗 Transformers 라이브러리의 이 객체들 중 push_to_hub()를 사용하여 Hub에 직접 공유할 수 있는 것은 무엇입니까?**
    1. 토크나이저(Tokenizer)
    2. 모델 설정(Model configuration)
    3. 모델(Model)
    4. 트레이너(Trainer)

6. **push_to_hub() 메서드 또는 CLI 도구를 사용할 때 첫 번째 단계는 무엇입니까?**
    1. 웹사이트에 로그인합니다.
    2. 터미널에서 'huggingface-cli login'을 실행합니다.
    3. 노트북에서 'notebook_login()'을 실행합니다.

7. **모델과 토크나이저를 사용 중인 경우, 이들을 Hub에 어떻게 업로드할 수 있습니까?**
    1. 모델과 토크나이저에 push_to_hub 메서드를 직접 호출하여 업로드합니다.
    2. 파이썬 런타임 내에서 huggingface_hub 유틸리티로 래핑하여 업로드합니다.
    3. 디스크에 저장한 후 transformers-cli upload-model을 호출하여 업로드합니다.

8. **Repository 클래스로 수행할 수 있는 Git 작업은 무엇입니까?**
    1. 커밋(Commit)
    2. 풀(Pull)
    3. 푸시(Push)
    4. 병합(Merge)
