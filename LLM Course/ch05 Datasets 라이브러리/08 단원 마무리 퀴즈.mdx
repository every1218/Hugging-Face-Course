2. **GLUE 작업 중 하나를 다음과 같이 로드한다고 가정해 봅시다:**

    ```py
    from datasets import load_dataset
    
    dataset = load_dataset("glue", "mrpc", split="train")
    ```
    **다음 명령어 중 어떤 것이 dataset에서 50개의 무작위 샘플을 생성할까요?**
    1. `dataset.sample(50)`
    2. `dataset.shuffle().select(range(50))`
    3. `dataset.select(range(50)).shuffle()`

3. **가정용 반려동물에 대한 pets_dataset이라는 데이터셋이 있고, 각 반려동물의 이름을 나타내는 name 열이 있다고 가정해 봅시다. 다음 접근 방식 중 어떤 것이 이름이 "L"로 시작하는 모든 반려동물에 대해 데이터셋을 필터링할 수 있도록 할까요?**
    1. `pets_dataset.filter(lambda x : x['name'].startswith('L'))`
    2. `pets_dataset.filter(lambda x['name'].startswith('L'))`
    3. `def filter_names(x): return x['name'].startswith('L')`와 같은 함수를 만들고 `pets_dataset.filter(filter_names)`를 실행합니다.

4. **메모리 매핑이란 무엇입니까?**
    1. CPU와 GPU RAM 간의 매핑
    2. RAM과 파일 시스템 저장소 간의 매핑
    3. 🤗 Datasets 캐시의 두 파일 간의 매핑

5. **다음 중 메모리 매핑의 주요 이점은 무엇입니까?**
    1. 메모리 매핑된 파일에 접근하는 것이 디스크에서 읽거나 쓰는 것보다 빠릅니다.
    2. 애플리케이션은 전체 파일을 먼저 RAM에 읽어 들일 필요 없이 매우 큰 파일의 데이터 세그먼트에 접근할 수 있습니다.
    3. 에너지를 덜 소모하여 배터리가 더 오래 지속됩니다.

6. **다음 코드가 실패하는 이유는 무엇입니까?**

    ```py
    from datasets import load_dataset
    
    dataset = load_dataset("allocine", streaming=True, split="train")
    dataset[0]
    ```
    1. RAM에 맞추기에는 너무 큰 데이터셋을 스트리밍하려고 합니다.
    2. `IterableDataset`에 접근하려고 합니다.
    3. `allocine` 데이터셋에는 `train` 분할이 없습니다.
